In questa sezione viene definita una funzione che permetter di caricare i dati di input e target, dividere il dataset in minibatch e applicare delle trasformazioni di preprocessing e data augmentation. In particolare, nella fase di data augmentation, verranno effetuate le seguenti operazioni:

  • rotazione in un range da -10 a 10 gradi
  • shift in altezza e in larghezza in un range da -0.2 a 0.2
  • zoom in un range da -0.2 a 0.2

def data_loader(preproc_function, fold, data_augment):
  if data_augment==True and fold=='train':
    pre_processing = ImageDataGenerator(preprocessing_function=preproc_function,
                                        rotation_range = 25,
                                        width_shift_range = .3,
                                        height_shift_range = .3,
                                        zoom_range = .3)
  else:
    pre_processing = ImageDataGenerator(preprocessing_function=preproc_function)

  if fold=='train':
    shuffle=True
  else:
    shuffle=False
  
  data_gen = pre_processing.flow_from_directory(directory=path+str(fold)+'/',
                                                  target_size=(224, 224),
                                                  color_mode="rgb",
                                                  batch_size=32,
                                                  class_mode="categorical",
                                                  shuffle=shuffle,
                                                  seed=1)

  return data_gen

4. Definizione del modello e fine-tuning¶

In questa sezione vengono definite delle funzioni in grado di applicare il processo di fine-tuning dato in input il modello pre-trainato (ResNet50, VGG16 e MobileNet) e successivamente addestrare tale modello sulle immagini importate precedentemente.
In particolare, verranno aggiunti i seguenti layer alla rete pre-trainata:

  • Flatten, in modo da ottenere un array mono-dimensionale

  • Fully Connected da 512 neuroni e funzione di attivazione reLu

  • Fully Connected da 256 neuroni e funzione di attivazione reLu

  • Output, con dimensione pari al numero delle classi (102 classi) e funzione di attivazione softmax

Per l'addestramento del modello finale, verr utilizzata come funzione di loss la categorical crossentropy e come ottimizzatore adam con learning rate=0.0001. Il valore di learning rate basso fondamentale quando si utilizzano tecniche di fine-tuning. Il numero delle epoche fissato a 100.
Infine, attraverso il campo callbacks del fit_generator(), stato possibile salvare i pesi dell'epoca migliore basandosi sui valori di log loss ed impostare un EarlyStopping con patient=10 basato sempre sui valori di log loss. Grazie a quest'ultima funzione, la fase di addaestramento terminer automaticamente nel caso in cui i valori di loss non crescano per 10 epoche consecutive.

def plot_history(history, model_name):
  fig = plt.figure(figsize=(18,6))

  fig.add_subplot(1,2,1)
  plt.plot(history.history['accuracy'], marker='.')
  plt.plot(history.history['val_accuracy'], marker='.')
  plt.title('model accuracy')
  plt.ylabel('accuracy')
  plt.xlabel('epoch')
  plt.legend(['train', 'test'], loc='upper left')

  fig.add_subplot(1,2,2)
  plt.plot(history.history['loss'], marker='.')
  plt.plot(history.history['val_loss'], marker='.')
  plt.title('model loss')
  plt.ylabel('loss')
  plt.xlabel('epoch')
  plt.legend(['train', 'test'], loc='upper left')

  fig.suptitle(str(model_name))

  plt.show()

def fine_tuning(img_rows, img_cols, color_type, pretrained_model):

  for layer in pretrained_model.layers:
      layer.trainable = False

  model = Sequential()
  model.add(pretrained_model)
  model.add(Flatten())
  model.add(Dense(512, activation = 'relu'))
  model.add(Dense(256, activation = 'relu'))
  model.add(Dense(102, activation = 'softmax'))

  model.summary() 

  model.compile(optimizer =  optimizers.Adam(learning_rate = 0.0001), loss='categorical_crossentropy', metrics = ["accuracy"])

  return model

def training_model(model_name, img_rows, img_cols, color_type, preproc_function, pretrained_model, data_augment):
  epochs = 100
  train_gen = data_loader(preproc_function, 'train', data_augment)
  test_gen = data_loader(preproc_function, 'test', data_augment)

  model = fine_tuning(img_rows, img_cols, color_type, pretrained_model)

  if os.path.isfile(os.path.join(path_drive+model_name+'.h5')):
    model.load_weights(os.path.join(path_drive+model_name+'.h5'))
    predict = model.predict_generator(test_gen)
    y_true = test_gen.classes
    y_pred = np.argmax(predict, axis=-1)
    f1 = f1_score(y_true, y_pred, average='macro')
    return model, f1

  else:
    callbacks = [ModelCheckpoint(os.path.join(path_drive+model_name+'.h5'), monitor='val_loss', save_best_only=True, verbose=0),
                 EarlyStopping(monitor='val_loss', patience=10, verbose=0)]
    history = model.fit_generator(train_gen, steps_per_epoch=len(train_gen), callbacks=callbacks,
                                  epochs=epochs, validation_data=test_gen, verbose=0)
    
    plot_history(history, model_name)

    predict = model.predict_generator(test_gen)
    y_true = test_gen.classes
    y_pred = np.argmax(predict, axis=-1)
    f1 = f1_score(y_true, y_pred, average='macro')
      
  return model, f1

8. Risultati e conclusioni¶

Per valutare le performance dei modelli verr utlilizzata la misura macro avarage f1-score. Di seguito un plot riassuntivo delle prestazioni dei modelli sulle immagini di test.

In [0]:
labels = ['ResNet50', 'VGG16', 'MobileNet']
x = np.arange(len(labels))  # the label locations
width = 0.15

f1score= np.around([resnet50_finetuned[1], vgg16_finetuned[1], mobilenet_finetuned[1]], decimals=4)
f1score_aug=  np.around([resnet50_finetuned_augment[1], vgg16_finetuned_augment[1], mobilenet_finetuned_augment[1]], decimals=4)

fig, ax = plt.subplots(figsize=(16,6))
f1_ = ax.bar(x - width/2, f1score, width, label='senza Data Augmentation', color='lightsalmon')
f1_aug = ax.bar(x + width/2, f1score_aug, width, label='con Data Augmentation',  color='steelblue')

def autolabel(rects):

    for rect in rects:
        height = rect.get_height()
        ax.annotate('{}'.format(height),
                    xy=(rect.get_x() + rect.get_width() / 2, height),
                    xytext=(0, 3),  # 3 points vertical offset
                    textcoords="offset points",
                    ha='center', va='bottom')
autolabel(f1_)
autolabel(f1_aug)

ax.set_ylabel('Scores')
ax.set_title('F1-score sulle immagini di test')
ax.set_xticks(x)
ax.set_xticklabels(labels)
ax.legend(bbox_to_anchor=(1, 1), loc='upper left', ncol=1)

fig.tight_layout()
plt.show()

Come possibile osservare dal grafico, le prestazioni dei modelli rimangono pressocch invariate sulle immagini di test, tranne per il modello VGG16 in cui vi un aumento di circa il 20% di $f_1$ - score. Invece, dalle curve di accuracy e loss, ottenute durante la fase di training dei modelli, possibile osservare che applicando la tecnica di data augmentation si riduce notevolmente l'overfitting, in cui si passa da una differenza di accuratezza tra training set e test set di circa il 10% senza data augmentation a poche unit con data augmentation. Quindi, a parit di prestazioni sul test set, vengono preferiti i modelli con in cui stata applicata la data augmentation e quindi "senza" overfitting, siccome risultano essere modelli pi robusti ed in grado di generalizzare meglio su nuove immagini.
